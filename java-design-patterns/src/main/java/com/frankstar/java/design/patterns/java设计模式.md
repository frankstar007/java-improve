### 抽象工厂模式的定义
**抽象工厂模式**：为创建一组相关或相互依赖的对象提供一个接口，而且无序指定
它们的具体类  

**优点**
- 封装性：每个产品的实现类不是高层模块需要关心的，只要知道工厂类是什么，就能
创建出一个需要的对象，省时省力
- 产品族内的约束为非公开状态

**缺点**
产品簇扩展非常困难，违反了开闭原则


### 使用场景
**使用场景**：一个对象族（或是一组没有任何关系的对象）都有相同的约束，则可以
使用抽象工厂模式。
